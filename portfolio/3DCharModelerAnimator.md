# 3D Character Modeler & Animator
3D Character Modelers & Animators create game-ready characters from concept to real-time implementation. 
This includes sculpting, retopology, UVs, texturing, rigging, animation, and demonstrating the character functioning in-engine with clean deformation and believable motion.

> [!IMPORTANT]
> ### What Your Portfolio Should Contain
> -   **Game-ready character models** (low-poly meshes built for real-time use)
> -   **High-poly sculpts** (for at least one character, showing detail and form)
> -   **Retopology + deformation-ready topology** (clean wireframes around joints and face loops)
> -   **UVs + baking + PBR texturing** (maps, UV layout, and texture breakdowns where applicable)
> -   **Material and shader work** (surface variety like skin/cloth/metal, shown in real-time rendering)
> -   **Rigged characters** (animation-ready control rigs and clean deformation tests)
> -   **Animation clips** (idle, walk, run, and an action animation; facial animation if applicable)
> -   **Gameplay usage footage** showing the character functioning in a game-style camera view
> -   **Concept art + turnaround sheets** (only if you created the concept)

# Portfolio Deliverables Checklist
Each character in your portfolio should be presented as a **complete character package**, the way it would be reviewed in a professional portfolio.

Use this checklist to organize your character page (or ArtStation post). **The goal is not to do every possible category for every character**, but to clearly show what work you personally completed.
-   Items labeled **Required** must be included for every character.
-   Items under **Add-On Deliverables** should only be included if you personally completed that part of the pipeline (sculpting, concept art, rigging, animation, texturing, shaders, etc.).
-   If you collaborated with someone, clearly label what you did vs. what someone else did.
-   If you use someone else‚Äôs concept art, you may include it for reference, but you must credit the original artist.

## Add-On Deliverables 

### D) Sculpting Pipeline
- [ ] **High-Poly Sculpt:** High detail sculpt showing anatomy/forms and surface detail.  
- [ ] **Sculpt Technique Breakdown:** Notes or callouts (ex: Dynamesh ‚Üí Subdiv workflow, alphas, VDM brushes).  
- [ ] **High vs Low Comparison:** Side-by-side images with polycount difference clearly shown.

### E) Concept Art + Turnaround Sheets 
- [ ] **Character Concept Sheet:** Final colored concept showing personality, mood, and style.  
- [ ] **Orthographic Turnaround:** Front / side / back views aligned for modeling reference.  
- [ ] **Design Iterations:** 3‚Äì6 rough sketches showing how the design evolved.  
- [ ] **Costume + Material Callouts:** Notes for key surfaces (cloth, leather, metal, skin, etc.).  
- [ ] **Prop/Accessory Callouts (If Applicable):** Closeups for weapons, bags, armor pieces, etc.

### F) Baking + Texture Deliverables
- [ ] **Bake Proof:** Close-up screenshots showing clean normal bake detail transfer.  
- [ ] **UV Layout:** Organized UVs with no major stretching and logical island grouping.  
- [ ] **Texel Density Notes:** Consistent density OR explanation of intentional differences.  
- [ ] **PBR Texture Maps:** Albedo/BaseColor, Normal, Roughness, Metallic (or Specular workflow).  
- [ ] **Optional Maps (If Used):** AO, Emissive, Opacity, Height.  
- [ ] **Texture Resolution Listed:** Example: ‚ÄúBody 2K, Head 2K, Accessories 1K.‚Äù

### G) Materials + Shader Work
- [ ] **Material Variety:** At least 2‚Äì3 distinct surface types (skin, cloth, metal, leather, etc.).  
- [ ] **Shader Demonstration:** Proof of specialized shading (skin SSS, hair anisotropy, cloth response, etc.).  
- [ ] **Material Breakdown:** A screenshot or page section showing which materials are assigned where.   
- [ ] üåü **Gameplay Real-Time Render:** Video showing textures/materials real-time rendered in game.  


### H) Rigging Deliverables
- [ ] **Skeleton Setup:** Joint hierarchy shown clearly (screenshot of rig in bind pose).  
- [ ] **Control Rig:** Controllers visible (IK/FK where appropriate).  
- [ ] **Rig Features:** IK legs, IK arms, and FK options where needed.  
- [ ] **Deformation Stress Test:** Screenshots or video of extreme poses (no collapsing/clipping).  
- [ ] **Stress Poses:** Arms overhead, deep crouch, elbow bend, shoulder bend, hip twist.

### I) Animation Deliverables
- [ ] **Idle Animation:** 5‚Äì10 seconds, clean readable silhouette.  
- [ ] **Walk Cycle:** In-place and/or forward moving version.  
- [ ] **Run Cycle:** In-place and/or forward moving version.  
- [ ] **Action Animation:** Attack / jump / dodge / interaction animation.  
- [ ] üåü **Gameplay Animation in Context:** Video of in-game character animations (Idle + locomotion and action if available)  
- [ ] **Rig-Visible Animation:** A version of each clip showing skeleton or rig controls.  
- [ ] **Timing/Beat Accuracy:** Animation hits clear beats (weight shifts, foot plants, impacts).

### J) Animation Process + Principles (Required for at least 1 animation)
- [ ] **Process Gallery:** From "Block-in" to "Splining" to "Polishing."  
- [ ]  üåü**In-Engine Animation System:** Simple state machine or blend tree shown.  
- [ ] **Principle Highlights:** Callouts showing Anticipation, Follow-through, and Timing/Spacing.  
- [ ] **Self-Analysis:** 3‚Äì5 sentences explaining what was adjusted to improve the motion.
