# 2D Game Artist
2D Game Artists create visual assets used directly in gameplay and UI, including characters, sprite animations, tilesets, icons, and interface elements. They focus on clarity, readability, and consistency, ensuring assets integrate smoothly into the game while supporting gameplay, narrative, and player experience.

> \[!IMPORTANT\]
> 
> ### What Your Portfolio Should Contain
>
> - **Concept Sketches & Color Studies**: Exploration of characters, props, or environment elements.
> -  **Silhouette / Readability Proof**: Ensures assets are recognizable at gameplay scale.
> -  **Sprite Sheets / Tilesets / UI Assets**: Game-ready files with proper layout, timing, and modularity.
> -  **In-Engine Usage**: Screenshots or videos showing assets functioning in a gameplay context.
> -  **Optional Specialized Assets**: Animated characters, parallax backgrounds, interactive UI, hero props, or modular environmental objects.
>     

# Portfolio Deliverables Checklist
Each portfolio piece should clearly show **your ability to create 2D assets that are readable, consistent, and functional in a game context**. Use this checklist to structure your submission.
-   **Required Deliverables:** Must be included for every portfolio piece.
-   **Add-On Deliverables:** Include only if you personally created them (iterations, parallax backgrounds, menu mockups, etc.).
-   If you collaborated, clearly indicate which parts you were responsible for.
